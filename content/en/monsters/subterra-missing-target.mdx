<Section icon={Target} title="Subterra Missing Target: What It Usually Means">

If **subterra missing target** is showing up on your screen, the game usually lost sync with the thing you are trying to track. That can be a quest marker, an interaction prompt, a combat reticle, or a layer objective. In practice, this is less about your save being broken and more about the client or server failing to refresh the target state.

The best way to solve it is to identify what kind of target disappeared, then apply the smallest reset that restores the tracker. If you treat every case the same way, you waste time. If you separate quest tracking from UI problems, the fix becomes much faster.

<Callout type="tip" title="Read the Signal First">
If the target disappears only after a teleport, treat it like a sync issue. If it disappears only in one area, treat it like a zone or quest-state problem.

<Section icon={Settings} title="Fast Repair Routine">

When the tracker breaks, do not start with a full reinstall or a long reset. Use a short repair order and stop as soon as the target returns. That keeps you in the run and avoids losing ore, loot, or quest momentum.

<Callout type="warning" title="Fix in Order">
Do not stack fixes at once. Change one thing, test the target again, and move to the next step only if the problem stays.
</Callout>

<Steps>
  <Step num="1" title="Reopen the objective or quest panel">
    Refresh the tracker first. If the game is holding an old objective state, opening the panel again often forces a clean update.
  </Step>
  <Step num="2" title="Leave and rejoin a new server">
    A fresh server is the cleanest way to clear desync. If the marker returns after a rejoin, the issue was probably session-based.
  </Step>
  <Step num="3" title="Re-equip your tool or weapon">
    Swap to another item and switch back. This helps when the target problem is tied to combat focus, interaction state, or input locking.
  </Step>
  <Step num="4" title="Return to the correct layer or NPC zone">
    Make sure you are standing in the right zone for the objective. Some targets only appear after you are physically in the right area.
  </Step>
  <Step num="5" title="Lower visual load and test again">
    If the UI still feels delayed, reduce graphics a little and test in a calmer area. Heavy effects can make the tracker feel unresponsive.
  </Step>
</Steps>

| Fix Order | Action | What It Solves | When to Stop |
|---|---|---|---|
| 1 | Reopen panel | Stale quest UI | Target returns immediately |
| 2 | Rejoin server | Session desync | Marker works in a fresh server |
| 3 | Re-equip item | Input or focus bug | Prompt or reticle comes back |
| 4 | Move zones | Wrong area / layer | Objective appears in the right place |
| 5 | Reduce load | UI lag or delayed refresh | Tracking becomes stable |

The key is to treat the tracker as a live system. If one quick reset restores it, you do not need a deeper fix. If none of the steps help, the problem is probably tied to the current server state, not your progress.

</Section>

<Section icon={Shield} title="Common Causes by Situation">

Most target problems fall into a few predictable buckets. Some are harmless UI hiccups. Others are caused by layer changes, fast travel, or joining a session after another player has already advanced the objective. The faster you identify the bucket, the less time you spend guessing.

<Callout type="info" title="Best Diagnostic Rule">
If the target fails in one place but not another, the problem is usually location-based. If it fails everywhere, the issue is usually a session or UI refresh problem.
</Callout>

| Cause | Typical Signs | Likely Fix | Risk Level |
|---|---|---|---|
| Quest not tracked | No marker, wrong objective text | Re-select the quest | Low |
| Server desync | UI feels normal, target stays missing | Rejoin a fresh server | Medium |
| Menu overlap | Marker hidden behind UI layers | Close panels and try again | Low |
| Wrong zone | Prompt never appears in current area | Return to the correct layer or NPC | Low |
| Input lock | Reticle or interaction fails after swapping items | Re-equip and test again | Medium |
| Client lag | Slow refresh, delayed prompts | Lower graphics and relaunch if needed | Medium |

If you are mining, be extra careful after teleporting between layers. Moving from one area to another can leave the tracker behind for a moment, especially if you open multiple menus at once. In that case, a clean re-track is often enough.

<Section icon={Sparkles} title="Best Prevention Settings for Longer Runs">
